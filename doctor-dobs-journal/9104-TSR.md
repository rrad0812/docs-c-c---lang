## Prekinite i ostanite rezidentni i novi projekat - Al Stivens

Prošlog meseca sam raspravljao o programiranju zasnovanom na događajima u C-u i napisao mali primer koji koristi tastaturu i miš za snimanje tekstualnih slika ekrana u datoteku na računaru. Program je koristan za izradu korisničke dokumentacije. Možete definisati pravougaonu oblast ekrana, snimiti je na disk i kasnije urediti pomoću uređivača teksta. Zatim ga možete odštampati ili spojiti u svoje datoteke za obradu teksta.

Primer programa se pokreće iz DOS komandne linije, što ograničava njegovu korisnost. Morate da pravite snimke ekrana u hodu dok program koji dokumentujete radi. Većina programa neće zadržati ekran dok izađete iz DOS-a da biste pokrenuli grabber ekrana, a čak i da hoće, DOS komandna linija bi iskoristila deo ekrana, eventualno skrolujući sa dela koji želite da snimite. Osim toga, posao prelaska između DOS-a i vašeg programa je smetnja. Stoga, da bi se uhvatili segmenti ekrana iz pokrenutih programa, program za hvatanje ekrana mora biti rezidentan u memoriji.

Ovomesečna kolumna vam pokazuje kako da pretvorite program sa komandne linije u rezidentan program (TSR). Tema TSR-a je do sada dobila sveobuhvatan tretman. Mnoge knjige i članci u časopisima su se bavili time. Sam sam obradio tu temu u tri knjige. Ponovo se obraćam jer sam želeo da završim program koji sam započeo prošlog meseca i zato što bi bilo nepravedno da vam bacim mnogo koda bez malo objašnjenja.

TSR je kludge. Postoji zato što dizajneri DOS-a nisu uspeli da predvide potrebu za jednostavnim prebacivanjem zadataka između istovremeno rezidentnih programa - ne multitasking, imajte na umu, već jednostavno prebacivanje zadataka. TSR je omogućen zato što su ti isti dizajneri morali da prikače spuler za štampanje na DOS za jednog korisnika i za jedan zadatak. Stavili su neke kuke u DOS kako bi omogućili njihovom štampanju spulera da radi u pozadini na način koji ne remeti integritet zadatka u prvom planu. Nisu objavili detalje tih udica, verovatno zato što je tehnika i krhka i ružna. Drugi programeri su obrnutim inženjeringom napravili kod DOS programa za štampanje spulera da bi napravili rane iskačuće uslužne programe kao što je SideKick. Na kraju, tehnike su postale opštepoznate, a TSR-ovi su se umnožili. Uzgred, možete da koristite operaciju Paste iz SideKickove beležnice da uradite ono što radi primer programa za hvatanje ekrana, osim što SideKick ne koristi miš da opiše deo ekrana koji želite da snimite.

Ovde ukratko -- odgovarajuće mesto za diskusiju o kludžu -- je kako funkcioniše iskačući TSR. Pokrećete TSR iz DOS komandne linije. On se vezuje za neke vektore prekida i koristi poseban DOS poziv da završi bez odustajanja od memorije koju zauzima. Otuda i naziv, "prekini-i-ostani-rezident". Kada pokrenete druge programe, DOS ih učitava u memoriju iznad TSR programa. Kada pritisnete preči taster TSR-a, rutina usluge prekida (koju je TSR prikačio vektoru prekida na tastaturi) presreće pritisak na taster i prolazi kroz nekoliko okretaja koji dozvoljavaju TSR programu da se pojavi.

TSR koji upućuje DOS pozive ne može se pojaviti u bilo koje vreme. DOS nije uvek u stanju da to prihvati. DOS se ne može ponovo prijaviti. Ako prekinete program dok je usred DOS poziva i pokrenete drugi program koji upućuje DOS pozive, DOS će se srušiti i spaliti. TSR mora da postavi indikator koji kaže da želi da se pojavi, a zatim da sačeka dok DOS ne kaže da je u redu. DOS to radi na dva načina. DOS održava nešto što se zove INDOS zastavica, indikator koji govori kada je DOS pokrenut i ne može se prekinuti. Sve dok je INDOS zastavica postavljena, ne smete da prelazite na drugi program koji upućuje DOS pozive. Kvaka je u tome što program COMMAND.COM upućuje DOS poziv da bi pročitao konzolu. Dokle god sedite na promptu komandne linije, DOS je pokrenut i INDOS zastavica je postavljena. Ako je INDOS obezbedio jedini način da vam dozvoli da prekinete DOS, ne bi se mogao pojaviti iskačući prozor dok ste bili na promptu komandne linije. Da bi zaobišao tu zamku, DOS dodaje još jedan kludž. Dok DOS petlja čeka na pritisak na taster, on povremeno postavlja stvari tako da dozvoli da bude prekinut. Zatim poziva broj prekida 0k28. TSR se vezuje za INT 28 i zna da je u redu da se pojavi kada se izvrši njegova INT 28 servisna rutina prekida.

Da bi sve ove stvari povezao, TSR pripaja prekid sa tastature kako bi pratio hitni taster, INT 28 prekid da bi pazio kada će DOS dozvoliti sebi da se pokrene, i prekid tajmera da bi sve pratio. Kada TSR servisna rutina za prekid tastature vidi da ste pritisnuli interventni taster, postavlja zastavicu koja to govori. Kada se rutina usluge prekida tajmera izvrši - 18,2 puta svake sekunde - gleda na zastavicu prečice. Ako je zastavica prečice postavljena, a INDOS zastavica nije, rutina usluge prekida tajmera iskače TSR. Ako se izvrši rutina INT 28 prekida i postavljena je hot flag, INT 28 rutina iskače TSR.

Pre nego što se pojavi, TSR mora da prebaci kontekst sa prekinutog programa na sebe. U uređenom okruženju za obavljanje više zadataka, operativni sistem upravlja prebacivanjem konteksta između zadataka. U TSR kludgeu, svaki rezidentni program mora to da uradi sam. Jedan od razloga što je situacija sa TSR-om krhka je taj što postoje različiti načini upravljanja promenom konteksta; ne radi svaki program na isti način i ne radi svaki program temeljno. Evo stavki konteksta koje morate promeniti. Morate da pređete sa steka prekinutog programa na stog TSR-a; morate da navedete DOS da pomisli da je TSR njegov jedini zadatak tako što ćete prebaciti njegov pokazivač na prefiks segmenta programa (PSP) pokrenutog programa; potrebno je da promenite adresu za prenos diska (DTA) iz one prekinutog programa u DTA TSR-a; potrebno je da privremeno onemogućite šta bi DOS uradio sa kritičnom greškom i kada korisnik pritisne Ctrl-Break ili Ctrl-C; ako će TSR koristiti miša, morate sačuvati trenutni kontekst miša. Možda ga koristi i prekinuti program.

U zavisnosti od toga šta radi TSR, morate se baviti i video režimom. Ako je TSR dovoljno pametan da prepozna trenutni video režim i koristi ga, ne morate da menjate video kontekst. S druge strane, ako je TSR program samo za tekstualni režim, na primer, potrebno je da snimite trenutni režim i, možda, sadržaj video memorije, i promenite video režim u onaj koji koristi TSR. Program za hvatanje ekrana ne brine o tome jer snima samo ekrane tekstualnog režima i pretpostavlja da ćete ga pokrenuti samo u tekstualnom režimu.

Rad sa PC video režimima nije trivijalan posao. Ako prekinuti program postavlja režime direktnim obraćanjem registrima CRT kontrolera, na primer, možda nećete moći da odredite trenutni režim, a još manje da ga sačuvate i vratite se na njega.

Nakon što izvršite sve to prebacivanje konteksta, možete pokrenuti TSR. TSR koji se izvršava kada je INDOS čist može slobodno da uputi skoro svaki DOS poziv. Onaj koji se izvršava iz svog INT 28 prekida može koristiti većinu DOS poziva iznad funkcije 0k0c. Funkcije od 0 do 0k0c su DOS pozivi ekrana i tastature i još uvek su podložne padu. Ne brinite. Ove funkcije su one koje većina TSR-a ne bi koristila zbog njihovog ponašanja i performansi. Većina TSR-ova će čitati tastaturu preko BIOS-a i pisati na ekran sa direktnim upisom u video memoriju. TSR-ovi treba da izbegavaju upućivanje poziva za dodelu DOS memorije ili pokretanje drugih programa iz funkcije DOS EKSEC.

Kada se TSR završi, mora da vrati kontekst na ono što je bio pre nego što se program pojavio. Imajte na umu da će program u ovom i prethodnom izdanju ispravno raditi kao TSR samo za verzije DOS-a veće od 3.0. DOS funkcije koje preuzimaju i postavljaju PSP nisu ispravno radile u ranijim verzijama DOS-a. Postoje trikovi za zamenu PSP konteksta u DOS verziji 2, a oni su opisani u knjigama pomenutim na kraju ove diskusije.

Koristi neke od Turbo C ekstenzija za čitanje i pisanje hardverskih registara i izvršavanje prekida. Drugi prevodioci imaju druge načine za obavljanje ovih nestandardnih operacija, tako da ako koristite kompajler koji nije Turbo C, morate da prenesete hardverski specifične stvari u konvencije vašeg kompajlera. Većina kompajlera nema analogiju segmentu Turbo C steka i pseudoregistratorima pokazivača steka, tako da ćete možda morati da napišete ekvivalent asemblerskog jezika da biste čitali i menjali vrednosti u SS i SP. Moj izlet u programiranje zasnovano na događajima i moja analiza TurboVision-a, Zinc-a i Mevel-a doveli su me do zaključka. Ako TurboVision, divan novi deo Turbo Pascal 6.0, nađe svoj put u Turbo C, on će bez sumnje biti C++ aditiv zbog svoje snažne orijentacije na klase. Korisnici C komponente Turbo C++, Turbo C 2.0 ili drugih C kompajlera neće imati koristi od TurboVision-a. Biblioteka cinka je takođe Turbo C++ proizvod. /*
 * A Terminate and Stay Resident (TSR) engine
 */

#include <dos.h>
#include <stdlib.h>
#include <stdio.h>
#include "mouse.h"
#include "keys.h"

void tsr_program(void);

#define KEYMASK  8
#define SCANCODE 52

extern unsigned _stklen = 1024;
extern unsigned _heaplen = 8192;

/* ------- the interrupt function registers -------- */
typedef struct {
    int bp,di,si,ds,es,dx,cx,bx,ax,ip,cs,fl;
} IREGS;

/* --- vectors ---- */
#define DISK    0x13
#define CTRLBRK 0x1b
#define INT28   0x28
#define CRIT    0x24
#define CTRLC   0x23
#define TIMER   8
#define KYBRD   9
#define DOS     0x21

unsigned highmemory;

/* ------ interrupt vector chains ------ */
static void (interrupt *oldtimer)(void);
static void (interrupt *old28)(void);
static void (interrupt *oldkb)(void);
static void (interrupt *olddisk)(void);

/* ------ ISRs for the TSR ------- */
static void interrupt newtimer(void);
static void interrupt new28(void);
static void interrupt newdisk(IREGS);
static void interrupt newkb(void);
static void interrupt newcrit(IREGS);
static void interrupt newbreak(void);

static unsigned sizeprogram; /* TSR's program size     */
unsigned dossegmnt;          /* DOS segment address    */
unsigned dosbusy;            /* offset to InDOS flag   */
static int diskflag;         /* Disk BIOS busy flag    */
unsigned mcbseg;             /* address of 1st DOS mcb */
static char far *mydta;      /* TSR's DTA              */

int hotkeyhit = FALSE;
int tsrss;          /* TSR's stack segment */
int tsrsp;          /* TSR's stack pointer */

/* -------------- context for the popup ---------------- */
unsigned intpsp;            /* Interrupted PSP address   */
int running;                /* TSR running indicator     */
char far *intdta;           /* interrupted DTA           */
unsigned intsp;             /*     "       stack pointer */
unsigned intss;             /*     "       stack segment */
unsigned ctrl_break;        /* Ctrl-Break setting        */
void (interrupt *oldcrit)(void);
void (interrupt *oldbreak)(void);
void (interrupt *oldctrlc)(void);

/* ------- local prototypes -------- */
static void resident_psp(void);
static void interrupted_psp(void);
static void popup(void);

void main(void)
{
    unsigned es, bx;
    /* ---------- compute memory parameters ------------ */
    highmemory = _SS + ((_SP + 256) / 16);
    /* ------ get address of DOS busy flag ---- */
    _AH = 0x34;
    geninterrupt(DOS);
    dossegmnt = _ES;
    dosbusy = _BX;
    /* ---- get the seg addr of 1st DOS MCB ---- */
    _AH = 0x52;
    geninterrupt(DOS);
    es = _ES;
    bx = _BX;
    mcbseg = peek(es, bx-2);
    /* ----- get address of resident program's dta ----- */
    mydta = getdta();
    /* ------------ prepare for residence ------------ */
    tsrss = _SS;
    tsrsp = _SP;
    oldtimer = getvect(TIMER);
    old28 = getvect(INT28);
    oldkb = getvect(KYBRD);
    olddisk = getvect(DISK);

    /* ----- attach vectors to resident program ----- */
    setvect(KYBRD, newkb);
    setvect(INT28, new28);
    setvect(DISK, newdisk);
    setvect(TIMER, newtimer);
    /* ------ compute program size ------- */
    sizeprogram = highmemory - _psp + 1;
    /* ----- terminate and stay resident ------- */
    _DX = sizeprogram;
    _AX = 0x3100;
    geninterrupt(DOS);
}

/* ---------- break handler ------------ */
static void interrupt newbreak(void)
{
    return;
}

/* -------- critical error ISR ---------- */
static void interrupt newcrit(IREGS ir)
{
    ir.ax = 0;            /* ignore critical errors */
}

/* ------ BIOS disk functions ISR ------- */
static void interrupt newdisk(IREGS ir)
{
    diskflag++;
    (*olddisk)();
    ir.ax = _AX;        /* for the register returns */
    ir.cx = _CX;
    ir.dx = _DX;
    ir.es = _ES;
    ir.di = _DI;
    ir.fl = _FLAGS;
    --diskflag;
}

/* ----- keyboard ISR ------ */
static void interrupt newkb(void)
{
    static unsigned char kbval;

    kbval = inportb(0x60);
    if (!hotkeyhit && !running)
        if ((peekb(0, 0x417) & 0xf) == KEYMASK)
            if (SCANCODE == kbval)    {
                hotkeyhit = TRUE;
                /* --- reset the keyboard ---- */
                kbval = inportb(0x61);
                outportb(0x61, kbval | 0x80);
                outportb(0x61, kbval);
                outportb(0x20, 0x20);
                return;
            }
    (*oldkb)();
}

/* ----- timer ISR ------- */
static void interrupt newtimer(void)
{
    (*oldtimer)();
    if (hotkeyhit && (peekb(dossegmnt, dosbusy) == 0) &&
            !diskflag)
        popup();
}

/* ----- 0x28 ISR -------- */
static void interrupt new28(void)
{
    (*old28)();
    if (hotkeyhit)
        popup();
}

/* ------ switch psp context from interrupted to TSR ----- */
static void resident_psp(void)
{
    intpsp = getpsp();
    _AH = 0x50;
    _BX = _psp;
    geninterrupt(DOS);
}

/* ---- switch psp context from TSR to interrupted ---- */
static void interrupted_psp(void)
{
     _BX = intpsp;
     _AH = 0x50;
    geninterrupt(DOS);
}

/* ------ execute the resident program ------- */
static void popup(void)
{
    running = TRUE;
    hotkeyhit = FALSE;
    intsp = _SP;
    intss = _SS;
    _SP = tsrsp;
    _SS = tsrss;
    oldcrit = getvect(CRIT);    /* redirect critical err  */
    oldbreak = getvect(CTRLBRK);
    oldctrlc = getvect(CTRLC);
    setvect(CRIT, newcrit);
    setvect(CTRLBRK, newbreak);
    setvect(CTRLC, newbreak);
    ctrl_break = getcbrk();     /* get ctrl break setting */
    setcbrk(0);                 /* turn off ctrl break    */
    intdta = getdta();          /* get interrupted dta    */
    setdta(mydta);              /* set resident dta       */
    resident_psp();             /* swap psps              */
    intercept_mouse();          /* intercept the mouse    */
    /* ------ save the video cursor configuration ------- */
    savecursor();
    normalcursor();
    unhidecursor();
    enable();
    tsr_program();              /* call the TSR C program */
    disable();
    /* ----- restore the video cursor configuration ----- */
    restorecursor();
    restore_mouse();            /* restore the mouse      */
    interrupted_psp();          /* reset interrupted psp  */
    setdta(intdta);             /* reset interrupted dta  */
    setvect(CRIT, oldcrit);     /* reset critical error   */
    setvect(CTRLBRK, oldbreak);
    setvect(CTRLC, oldctrlc);
    setcbrk(ctrl_break);        /* reset ctrl break       */
    disable();
    _SP = intsp;                /* reset interrupted stack*/
    _SS = intss;
    running = FALSE;            /* reset semaphore        */
}
